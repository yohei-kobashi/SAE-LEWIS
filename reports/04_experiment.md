  系譜だけでは単独帰属不可。
- **単一要因ablation確定(2026-07-21、v5f_noS3 = v5fレシピからFLOW_INITのみ
  除去、復唱枠、probe500)**: **true 0.0361 / random 0.0060 / 同枠steer
  0.1804(不変)**。λ-IoU 0.3717(true) vs 0.1836(empty)。
  → EF基盤warm startの除去だけでv5fの0.2485が**85%崩壊**(steerすら
  下回る)。EF基盤は本手法の最重要要素であることが単一要因で確定。
  bare枠の系譜証拠(v1-v3全滅)と合わせて帰属が閉じた。
- **Tier1-③ noctr確定(2026-07-21、対照教師なし: MM_PROB=0, EMPTY_PROB=0)**:
  true **0.2104** / random 0.0000 / empty copy 0.99。−15%の劣化に留まり、
  random特異性と前提保護は構造的に維持 — 対照教師は「精度の上積み」
  要素で、EF基盤(−85%)ほどの必須性はない。
  - **λ診断で本当の寄与が見える**: empty条件のλ-IoUがv5f 0.1024 →
    noctr **0.4049** — 対照教師なしではλが「スペックなしでも破損位置を
    指す近道」を残す(発火量はほぼ0のため行動はコピーに留まるが、
    WHERE場のスペック特異性が失われる)。trueのλも弱く甘くなる
    (発火0.318→0.289、IoU 0.538→0.487)、random時のコピー崩れ増
    (copy 0.9459→0.9118)。→ 対照教師の役割は (a) true精度の上積み
    (相対+18%)と (b) **λ場のスペック特異性**(解釈可能性主張の根拠)。
    行動防護(empty copy 0.99)自体はノルム予算+復唱枠コピー事前が担う
    (④nobudgetで裏取り予定)。
- **Tier1-④ nobudget確定(2026-07-21、ノルム予算なし: NORM_REG_W=0,
  NULL_NORM_W=0)**: true **0.3166(v5f比+27%)** / random 0.0000 /
  empty copy 0.99 / sim_target 0.7660。**劣化ではなく大幅改善**。
  - random時のコピー維持はむしろ向上(0.9679 vs v5f 0.9459)、λはempty発火
    が僅かに漏れる(mean 0.083、IoU 0.22)が行動は完全にコピー。
  - **解釈**: 行動防護(empty/random時の無編集)はノルム予算ではなく
    対照教師+復唱枠のコピー事前で足りており、予算は編集容量を
    制限して真の編集を27%犠牲にしていた。③noctrと合わせ、
    「対照教師=特異性の担い手 / ノルム予算=容量の税」と役割が確定。
  - ユーザー決定(2026-07-21): **nobudgetをチャンピオンに採用**(exact優先)。
  - **先行研究上の位置づけ(一次資料逐語確認済み 2026-07-21)**: ノルム予算の
    除去は「標準的設計への回帰」として引用で支えられる。
    (a) **ReFT/LoReFT**(Wu+, NeurIPS 2024)— residual streamへの学習介入の
    代表系は、目的関数が**教師強制cross-entropyのみ**(式(4):
    min_ϕ{−Σ log p_Φ(y_i|x,y_<i)})で、**介入出力Δhへのノルム正則化を
    一切持たない**(構造制約はRの直交性のみ、DiReFTはそれすら除去)。
    (b) **AxBench**(Wu+, PMLR v267)— 介入強度は事前に制約せず結果で選ぶ:
    "We select the optimal steering factor for each method independently
    for every concept based on which factor achieves the highest overall
    steering score"。彼らの最良表現手法ReFT-r1の正則化もL1(非top-k
    latentの疎性)であり**Δhノルム制約ではない**。
    → 分野の実践は「介入の大きさは目的関数/結果に委ねる」。予算(0.5·
    ‖dvec_L‖)は我々が足した保守的追加であり、その税(−27%)を§5bの
    ablationが定量化した、と書ける。直接「ノルム罰の除去で改善」を報告
    する先行研究は無い(ReFT系はそもそも持たない)ため、**「除去が良い」
    ではなく「持たないのが標準で、持つコストを我々が実測した」**と表現
    すること。
  - **予算を「入れる」側の先行研究(逐語確認済み 2026-07-21)— 当初設計の
    根拠として引用可能**:
    (c) **LEACE**(Belrose+, NeurIPS 2023)— 表現への介入は最小摂動で
    行うべしという系譜の代表。目的関数そのものが摂動ノルム最小化:
    "argmin E[‖PX−X‖²_M] subject to Cov(PX,Z)=0"、動機は "we'd like to
    make a 'small' edit to X so that useful information contained in X is
    maximally preserved"。大きな編集の巻き添え損害(INLP比のperplexity
    悪化)も実測している。
    (d) **Householder Pseudo-Rotation**(Pham & Nguyen, EMNLP 2024)—
    活性ノルムの一貫性を壊す加算編集への直接警告: "If the scaling factor
    is too large, the additive edit might drastically alter the activation
    norms in each layer, violating the norm consistency property of LLMs.
    In extreme cases, this change can lead to the generation of complete
    gibberish"。同時に「ノルムを守ると編集力が落ちる」ジレンマも明言
    ("If the scaling factor is set too low ... limited abilities to
    shift an activation")。
    → **論文の枠組み**: ノルム予算は(c)(d)の「最小摂動・ノルム一貫性」系譜
    に基づく設計であり、(a)(b)の「タスク損失に委ねる」系譜と対立する。
    我々のablationはこの対立に**編集タスクでの実測**を与える: L12では
    HPRの言うジレンマの「編集力側」が支配的(予算の税−27%)、L4/L20
    (40k)では「一貫性側」が支配的(予算が保護、−15%/−24%)— つまり
    **どちらの系譜が正しいかは層(信号強度)依存**、という新しい知見に
    昇格できる。
  - **⚠️ 層依存性(2026-07-21夜、L4/L20のnb 40k完了で判明)**: nobudget
    40kは **L4 0.1142(予算あり0.1343より−15%)/ L20 0.0321(0.0421
    より−24%)** — 予算除去の利得はL12(+27%)に特異的で、浅層・深層
    では予算が正則化として働いていた可能性。特異性・防護は全層維持。
    最終判断は80k延長(実行中)の完了後。層別にレシピが分かれる場合は
    「層ごとにablationで選択した」と開示する。
  - **L12 nb 80k完了(2026-07-21夜)**: true ef exact **0.2886** —
    40kの**0.3166を下回り、勝者は40k**。予算ありL12でも延長が悪化させた
    (0.2485→0.1924)のと同型で、「収束済みモデルへのLR再始動延長は有害」
    がnobudgetレシピでも再現。**nobudgetチャンピオン=L12 40k(0.3166)で
    確定**(旧oracle-specプロトコル値)。nbqキューはW=40000のため
    ablation学習(noS3/noctr)は40kのみ実行。
  - **L20 nb 80k完了(2026-07-22未明)**: true ef exact **0.0441**
    (nb 40k 0.0321から+37%だが、予算あり80k 0.0601には届かず)—
    L20は40k/80kどちらの学習量でも予算あり優位。※旧oracle-spec
    プロトコル(レシピ比較の診断として有効。主表は新プロトコルで再測定)。
  - **L4 nb 80k完了(2026-07-22未明)**: true ef exact **0.2846**
    (nb 40k 0.1142の2.5倍、予算あり80k 0.1804を+58%超過、random 0.0020 /
    empty copy 0.99維持)— **層依存性の解釈を修正**: L4の nb 40k劣化
    (−15%)は「予算が保護」ではなく**未収束**だった。十分学習すれば
    L4もnobudget優位(L12と同方向)。予算あり優位が残るのは**L20のみ**
    (信号最弱層)。nbレシピの層プロファイル(80k時点): L4 **0.2846** /
    L12 0.2886(80k; 40kなら0.3166)/ L20 0.0441 — 浅層が中間層に並ぶ
    大幅な絵の変化(旧oracle-specプロトコル値、付録/レシピ選択用)。
  - **nbレシピ版ablation完了(2026-07-22、nbqキュー完走、旧プロトコル)**:
    nb_noS3 true **0.0581**(nbチャンピオン0.3166比−82% — EF基盤の必須性

    0.0020・empty copy 0.99は健全維持。予算あり時のnoctr(−15%)と符号が

|---|---|---|

  (§5dの「ef=steer」はL4固有の限界ではなく学習量の問題)。
- bestが78kまで更新され続けた点は更なる延長余地を示唆。

**考察 — AxBench準拠がほぼrandom並みである理由**:
(i) **読み取り器≠書き込み器**: AUROC選択は相関基準で「検出に最適な
latent」を選ぶが、検出と生成の因果は解離する — P-Jで単独観測済み
(AUROC r1: 検出0.939なのに介入効果ゼロ)。feature分裂により因果は
同点クラスタに分散し、1本の押し上げは並行経路が補償(LinguaLens自身の
補償観測と同機序)。
(ii) **粒度不足**: 元タスク(自由生成のどこかに概念が出ればよい操舵)
向けの大域的定数加算は、exactの連言条件(編集∧正位置∧他無傷)を
系統的に満たせない。
(iii) **動作点の窓が無い**: 用量反応はある(f0.2→1で0.120→0.166、
f4で0.036に崩壊)が、「反転に足る強さ」と「保存できる強さ」の窓が
ほぼ交わらない(steer α崖の狭隘版)。
(iv) **床の先取り**: amp床0.120は素のrewriterの文法修復 — 鈍い介入でも
取れる簡単ケースは介入なしで取れ、残りは語彙固有編集でカテゴリ粒度の
1方向には原理的に載らない。
→ 裏付け: 同じ加算機構でも64本・符号つき・大きさ保存のspecを使う
機構対応steerは明確な効果(L12 amp 0.118、FIC 0.25-0.84)。律速は機構
でなく**介入信号の情報量**。AxBenchの結論(SAE操舵<prompting、例外は
学習介入ReFT-r1)とも整合。

  α∈{1,2,3}の小グリッド+最低k床(例8)を併用。
