# 05 — Analysis / Discussion / Limitations 執筆資料(⚫EF除外後の改訂版)

## 1. Analysis の物語

### 1a. 3層分解 — 3ドメイン一致の乖離(束ねる解釈枠)

> **検出できる ≠ 因果的に担う ≠ 介入で書ける**

| 層 | 証拠 | 数値 |
|---|---|---|
| 検出 | AUROC選択器(sanity gate) | top-1 mean 0.939(彼らの0.917超え) |
| 因果 | P-J: そのtop-1が介入で動かない / FRC3は微弱純粋 | 0.86× / 67:0 |
| 実行 | P-I WHAT不能 + 判別木C類21現象 | readout exact 床 / WHERE陽性でも介入編集0 |

同型の乖離の先行と並置: **Hase et al.**(重み編集: localization≠editing)、
**AxBench**(steering: 検出+32%でもsteering不変)、我々(テキスト編集)—
**3ドメインで一致**して観測された乖離、という強い構成で書く。
系譜: amnesic probing(decodable≠used)の因果版。

### 1b. 局在性スペクトルの読み方(免許規則)

- **操作幅**(B-1 k掃引): 「因果的操作に何本要るか」— 事例内容が混入する
  ため**表現幅とは書かない**。
- **形1**(P-J r掃引): target-freeなので内容混入なし → 「因果的に有効な
  現象表現はtop-3より広い」(定義取替を明示)。
- **形2**(S_min安定核×FRC3): 事例間共通=現象側、入れ替わり=内容側の
  操作的分解 → 安定核とFRC3の交差・サイズ比較で「対応集合はtop-3より
  多い/別物」が初めて書ける。
- 対抗仮説の処理: top-1 steering(arXiv:2510.01246「top-1で足りる」)は
  **評価器が違う**(生成品質 vs 厳密編集)— 「評価器が局在性の結論を
  変える」として両立させ、それ自体を(e)の論拠に転化。

### 1c. FRRとexactの緊張関係
steerのFRR(方向実現)とexact(正確な着地)は別軸 — net-FRR 0.41に対し
feature別では負のnet(expressive −0.40等)が「見かけの実現=偽陽性」を暴く。
多軸報告の必然性はP-Nの指標間順位反転too。

## 2. Limitations(順序もこのまま推奨)

### 2a. 🔴 最重要 — 仕様の出所
事例レベル仕様は実際の(src,tgt)から計算した特徴差分で、運用時にtargetは
無い。「仕様を概念から得る」のは概念→latent引き当て(AxBenchが0.695と
測った当のもの)。主張の射程 = 「**仕様が与えられたときの因果的実現**」。
実例: AxBenchのベンチにはtargetが無いため我々の仕様が定義できず、
相互評価に我々の腕を置けない(axbench_testdata.md §3)。
部分的回答: P-J r=64 — target-freeでも因果情報は事例の半分弱まで回収可能。

### 2b. 判別木の不定性
切り分けは示唆に留まる: C類(効果器側)はWHERE陽性+介入不可で定義したが、
「活性に内容情報が無い」ことの直接証明ではない(WHEREは位置の証拠)。
D類はLM側かSAE側か不定。B類は2現象のみで一般化しない。

### 2c. 測定器の限界
- S_minは貪欲探索の**上界**(全部分集合は探索していない)。
- FRRのjudgeはreliabilityのみ検証(validityは人手なしに主張しない)。
  減衰の代数(下記3)は非差異的誤分類が前提で、per-system自己一致が
  その検査を兼ねる。
- 効果器2種(steer/clamp)のみ — 介入編集の失敗が「あらゆる介入の失敗」
  とは言えない(C類の帰属は効果器相対)。

### 2d. その他(1文ずつ)
英語のみ・Gemma-2-2B/層12単一スケール・LinguaLens minimal pairsに限定。
LinguaLensのFRCは論文(条件付き)とコード(周辺)が乖離 — コード側に忠実。
再現アンカーはパターンとレンジ(LinguaLensはモデルが違う。AxBenchは
同一スタックだがmax_actの出所と概念数が差)。

## 3. 減衰(attenuation)の書き方テンプレ

1. 対応差はアフィン縮小: p_A−p_B = (θ_A−θ_B)(q0+q1−1) → 符号保存・
   保守的下界(judgeがチャンスより良い場合)。
2. 前提 = 非差異的誤分類。システム間でjudge難度が違いうるので無料でない。
3. per-system自己一致率がその検査(1測定2役)。
4. 反例(Jurek 2005)は多値・相関・小標本由来 — 2値×非差異×対応ありでは
   代数が成立、と1文で切り分け。
5. Chen et al.(arXiv:2601.05420)の補正は校正集合が要る — 我々は符号と
   下界だけが必要。
