# Part I — 理論的背景

## 1. 研究の目的と貢献の位置づけ

- **前提(目的)**: LinguaLensと同様、言語学的featureに対応するSAE
  activationsを見つけること(新たな抽出手法は提案しない)。
- **提案システム**: 同定された活性(仕様)を信号として受け取り、編集を
  **Δhとしてresidual streamに描画する学習editor(Intervener)**。
  テキストは凍結LMの生成として出る = do(residual)の介入(ReFT類縁)。
- **中心主張**: activationsの因果妥当性は**編集によって**評価されるべき
  であり、editorによる編集の成立がその因果証明。評価の幅: (i) exactと
  いう厳密なテキスト接地基準、(ii) 介入本数の局在性スペクトル、
  (iii) net-FRRの特異性、(iv) 現象別の到達層(判別木)。
- 貢献文(R5で確定): 「事例レベルSAE仕様に条件付けられた介入生成器を
  提案し、CausalGymの言語学的因果評価を**テキスト編集の実行**まで拡張、
  SAEBench系の介入評価に**編集という評価器**を加える」。

## 2. 条件付けと介入 — 分水嶺は**出力インターフェース**

| | 条件付け(トークン出力EF/B2) | 介入(steer/clamp/**Intervener**) |
|---|---|---|
| 出力 | **トークン列**(編集器が文を書く) | **Δh(embedding)** — 文は凍結LMが書く |
| 因果式 | P(edit \| Z=z) — zは**入力** | P(Y \| do(Z=z)) — 活性側を**書き換える** |
| 検証器 | 学習された編集器(**内生的** — その符号を読むよう最適化済み) | **凍結LM自身(外生的** — テキストは凍結LMの計算が生む) |
| 証明できること | zが編集を特定する**情報**を持つ(読み手相対) | LMの計算が**その方向に応答する**(因果) |

- 判定基準はアーキテクチャではなく**出力インターフェース**(2026-07-17
  決定): 双方向エンコーダや特徴トークン条件付け(EFの機構)は
  Intervenerに流用してよい — トークンを直接出力する構成だけが条件付け。
- S1(hazard)/S2(特徴トークン)/S3(局所化)のトークン出力EF系は
  因果証明にならず論文から除外(コード・runsは履歴として残す)。
- Intervenerの内容はReFT同様「学習される」が、出力チャネルがΔhである
  限りテキスト生成は凍結LMの計算 = 検証器は外生的。仕様への特異性は
  random/empty統制とゼロショットOOD、介入サイズはノルム予算が担保。

## 3. バックドア基準と「外生的検証器」問題

- 評価でZ(仕様)は**こちらが設定**する(true/random/empty割付)=入力への
  正真正銘の do(Z)。**バックドア基準は満たす** — Zへの交絡パスは無い。
- 問題は識別ではなく**推定量の帰属先**: 識別されるのは「**この学習済み
  編集器を含むシステム**でZが編集を起こす」であって、「ZがLM/SAE幾何に
  おいてfeatureに対応する」ではない(輸送可能性・構成概念妥当性の問題)。
- 学習された読み手は**単射に近い符号なら何でも**読めるようになる —
  効果の存在が符号(SAE)の性質を弁別しない。
- **SAEのdecodeとの違い**: W_decは再構成のみ学習した**基底変換**で、
  出力は凍結LMの下流計算に食われる(検証はLMが行う)。編集器のdecodeは
  **タスク解読器**(成功するように最適化される)。
- 実例(自前の実測): universal_quantifiers の AUROC=1.000 latent
  (ラベル "proper names and titles")— 読み手ベース評価はこの相関latentを
  「対応」と誤認しうるが、介入は暴いた(0.86×、因果ゼロ)。
- **Intervenerへの適用(なぜ学習editorでも帰属が守られるか)**:
  (i) 出力はΔhのみでテキストは凍結LMが生成(検証器は外生的)、
  (ii) ノルム予算で「介入サイズ」をsteer描画と同桁に拘束(自由な
  soft-prompt符号チャネル化を防ぐ)、(iii) v2はΔhをα·dvec基底+補正に
  分解 — 基底はSAE幾何(W_dec)そのもので、学習が動かすのは補正のみ、
  (iv) random/empty統制とゼロショットOODが仕様特異性を担保。

## 4. 操作ハンドル ≠ featureに対応するactivations(免許規則)

- **操作ハンドル** = 介入で編集が動く集合。feature方向に加えて**事例の
  内容(WHAT)・整合性維持が混入**する(実測: M0/P-Bでrank17-32が方向と
  内容を分離不能に符号化)。
- したがって「事例レベルkで編集成功 → featureの表現はk本」は**書けない**。
  書けるのは「因果的**操作**にはtop-3では足りない」(操作インターフェース
  幅)まで。
- 「対応する活性はtop-3より多い/別物」を書ける正規ルートは2つ:
  - **形1: 現象レベル(target-free)r掃引** = P-J。現象リストは評価に使わ
    ない別ペア群の統計から作るので**事例内容が構成上混入しない**。実測は
    r単調成長(FRC r=3: 0.17 → r=64: 2.51、常にtrue専有)・64で非飽和 →
    「因果的に有効な現象表現はtop-3より広い」。ただし「対応」の定義を
    検出的(発火統計)→**因果的(介入で動く)**に取り替えたと明示 — この
    取替の必要性こそ本研究の主張の実演(AUROC 0.939のtop-1が因果ゼロ)。
  - **形2: S_minの事例横断安定核**。事例間で共通に生き残るハンドル=現象、
    入れ替わる部分=内容、という操作的分解。安定核をFRC top-3と本数・
    中身で直接比較(compare_smin_frc.py)。

## 5. 3層分解(3ドメイン一致の乖離)

> **検出できる ≠ 指令できる ≠ 介入で書ける**

- 検出: AUROC top-1 mean 0.939(AxBench自身のSAE-A 0.917超え)
- 因果: その完璧なtop-1が介入では動かさない(0.86×)/ FRC3は微弱純粋(67/0)
- 実行: 介入編集が成立する現象は71/98(判別木A類)、WHERE陽性でも介入が
  実行できない現象21(C類=WHAT問題)
- 同型の乖離の先行: Hase et al.(重み編集: localization≠editing)、
  AxBench(steering: 検出+32%でもsteering不変)、我々(離散編集)—
  **3ドメイン一致**として書ける。

## 6. 情報軸(条件付け評価)の扱い — 現状不採用、復活条件つき

- 対応関係(z入力→編集器→出力)の評価は「**情報的忠実性**」として成立
  し得るが、SAEの手柄と主張するには**random辞書対照**(同一レシピで辞書
  だけランダムに差し替えて再学習)が必須。
- 結果A(SAE≫random): 「SAE幾何は編集情報を転移可能な形で担う」(情報軸)。
  結果B(同等): 「読み手ベース評価は辞書を弁別しない → だから介入評価が
  必要」= 主主張のablation的補強。**どちらでも紙面に載る**が、コスト
  (学習~7h+probe)と焦点の判断で現状は不採用。
- 選ばれた活性の正式な呼び方: 「**介入によって因果的に検証された操作
  ハンドル(causal handles)**」。条件付けで読めるという第二の性質は
  主張しない。

## 7. 用語と免許の一覧

| 書いてよい | 書いてはいけない |
|---|---|
| 「同定活性への介入が編集を実行(因果証明)」 | 「EF/routedの成績」を論文に載せる(⚫) |
| 「因果的操作にはtop-3では不十分」 | 「事例k=32がfeatureの表現幅」 |
| 「因果的に有効な現象表現はtop-3より広い(r掃引・定義取替を明示)」 | 検出的な意味での「対応」とすり替える |
| 「介入評価に編集という評価器を加える」 | 「介入ベースSAE評価は初」(SAEBench先行) |
| 「reliabilityを測る」(judge) | 「judge品質/validityを測る」 |
