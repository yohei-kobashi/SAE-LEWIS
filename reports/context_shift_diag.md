# 診断7 — 同定文脈と注入文脈のSAE特徴シフト(査読対応、2026-07-19)

**問い**(ユーザー指摘): 仕様はプロンプトなしの裸の文で同定するが、介入は
復唱プロンプト内のsrc位置で行う。両文脈のSAE特徴は食い違わないか。

**測定**(scripts/diag_context_shift.py、L12、n=100、BOS除外プール):

- A = 裸のsrc × ベースgemma-2-2b(同定条件)
- B = 裸のsrc × gemma-2-2b-it(モデル差の統制)
- C = 復唱プロンプト内のsrcスパン × -it(注入条件)

| 指標 | 値 |
|---|---|
| pooled top-64 Jaccard A↔C(**総シフト**) | 0.353 |
| pooled Jaccard A↔B(モデル差のみ) | 0.691 |
| pooled Jaccard B↔C(文脈差のみ) | 0.401 |
| **Aのtop-64がCのtop-64に残存(spec残存率)** | **0.518** |
| 位置別top-16 overlap B↔C | 0.590 |

## 読み

1. **同定した特徴の約半分(残存率0.52、位置別0.59)は注入位置でも上位の
   まま**。シフトは実在するが崩壊ではない。
2. 分解すると、モデル差(base→it、0.69)と文脈差(裸→プロンプト内、0.40)
   の両方が寄与し、**文脈差の方が大きい**。
3. 介入タイプ別の含意:
   - 方向加算型(A2 steer、A4学習Δh): W_dec幾何は文脈非依存なので操作の
     意味は保たれ、ずれは動作点のみ — 頑健
   - 値設定型(A1 LinguaLens式clamp): 裸文脈の絶対値をsetするため最も
     感受的 — A1の解釈に本診断を注記
4. 実証面: この不一致構成のまま rewrite枠steer 0.2385、bare枠v4 editor
   0.1904、復唱枠steer 0.1563 が成立しており、編集可能性は保たれている。
5. 位置づけ: 「裸のペアで同定し、タスクプロンプト下でbase学習SAEを
   instructモデルに適用して介入する」という構造は**LinguaLens自身の
   プロトコルと同一**であり、本研究固有の逸脱ではない(論文にこの1文と
   上表を注記として置く)。

## 追試: プロンプト語順の入れ替え(ユーザー提案、2026-07-19採用)

因果LMではsrcトークンは左文脈にしかattendしないため、指示文をsrcの
**後ろ**に置く(src先行)と注入文脈が同定文脈に近づくはず、という仮説を
検証した:

| 指標 | 指示先行 | **src先行(採用)** |
|---|---|---|
| 素モデル復唱率 | 99% | **99%** |
| spec残存率(A→C) | 0.518 | **0.591** |
| 位置別top-16 overlap(B↔C) | 0.590 | **0.703** |
| 文脈シフトJaccard(B↔C) | 0.401 | **0.501** |

全指標が改善し復唱率は同一 → **src先行を採用**
(REPEAT_PROMPT = "Input: {src}\n\nRepeat the input sentence exactly.
Never output anything else.")。出典: runs/prod_gemma_v6/diag7_srcfirst/。

## 既知の注意

- 初版測定はA/BのプールにBOS位置を含めており、attention-sink活性が
  top-64を支配して見かけ上Jaccardが崩壊した(0.015)。実specパイプラインは
  編集局所プール(BOSフリー)なのでBOS除外が正しい比較(修正済み)。

出典: runs/prod_gemma_v6/diag7_context_shift_v2/report.md

## 追記(2026-07-23): この診断の解決策が本線に昇格

診断7が定量した「同定(裸文)と注入(復唱プロンプト文脈)の測定点ずれ」
は、**⑦文脈内spec構築**(同定測定を復唱プロンプト内のgemma-2-2b-it残差・
srcスパン位置で行う、04§9m)により解消され、exactが両方向+15〜16%改善
した(ablation 0.148)。⑦は最終構成の採用候補 — 本診断は「ずれの定量」
から「⑦の動機付け」として論文に引く。
